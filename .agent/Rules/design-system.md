# Design System Rules

## Product Personality

SecureGate should feel:
- secure
- modern
- calm
- trustworthy
- minimal

The UI should communicate confidence and safety.

---

# Visual Direction

Use:
- clean layouts
- generous spacing
- subtle contrast
- minimal decoration
- soft borders
- focused interactions

Avoid:
- overly playful UI
- excessive gradients
- visual clutter
- loud colors

---

# Color Direction

Primary:
- Neutral dark tones
- Deep blue accents
- Slate grays

Feedback:
- Green for success
- Amber for warnings
- Red for errors

---

# Typography

Use strong hierarchy:
- large headings
- readable body text
- clear labels
- subtle helper text

---

# Form Design

Forms must:
- feel simple
- show validation clearly
- support accessibility
- provide loading feedback

---

# Authentication UX

Authentication flows should:
- reduce confusion
- avoid overwhelming users
- explain actions clearly

Examples:
- verification pending
- reset link sent
- invalid token
- session expired

---

# Buttons

Buttons should:
- have clear hierarchy
- include disabled states
- include loading states

---

# Inputs

Inputs must:
- have labels
- have focus states
- support keyboard navigation
- show validation errors clearly

---

# Dashboard

Dashboard should:
- remain intentionally minimal
- focus on authenticated state
- communicate successful access

Avoid feature creep.

---

# Responsive Design

The UI must:
- work on mobile
- work on tablet
- work on desktop

---

# Accessibility

Always:
- use semantic HTML
- support keyboard users
- maintain contrast ratios
- include aria labels where necessary
